# GitHub Cloud Agent: User Manual

This skill enables **Server-Side Autonomous Coding**. By setting this up, your repository monitors issue comments and deploys an AI agent to write code, fix bugs, and open Pull Requests automatically, 24/7.

## ðŸ“¦ Installation

There are two parts to installing this:
1.  **Scripts**: The "Brain" (Python scripts).
2.  **Workflow**: The "Trigger" (GitHub Actions YAML).

### Step 1: Install Skills Submodule
If you haven't already, add the Skills repo as a submodule to your project:
```bash
git submodule add https://github.com/PCHANUL/Skills.git .skills
git commit -m "Add Skills submodule"
```
*This places the logic in `.skills/github-cloud-agent/scripts/`.*

### Step 2: Deploy the Workflow Trigger
GitHub Actions **must** verify the workflow file exists in your repository's `.github/workflows/` directory.

Copy the provided template:
```bash
mkdir -p .github/workflows
cp .skills/github-cloud-agent/workflows/agent.yml .github/workflows/cloud-agent.yml
```
*Note: You may need to edit the path in `cloud-agent.yml` if your submodule path is different from `.skills`.*

### Step 3: Configure Permissions
1.  Go to your Repository on GitHub.
2.  **Settings** > **Actions** > **General**.
3.  Scroll to **Workflow permissions**.
4.  Select **Read and write permissions**.
5.  Check **Allow GitHub Actions to create and approve pull requests**.
6.  Click **Save**.

### Step 4: Add API Key Secret
The agent needs a brain (LLM).
1.  Get an API Key from [Google AI Studio (Gemini)](https://aistudio.google.com/) or [Anthropic (Claude)](https://console.anthropic.com/).
2.  Go to **Settings** > **Secrets and variables** > **Actions**.
3.  Click **New repository secret**.
4.  **Name**: `LLM_API_KEY`
5.  **Value**: `Paste-Your-Key-Here`
6.  Click **Add secret**.

## ðŸŽ® Usage Guide

### The Magic Command: `/agent`

Once installed, the agent listens to **Issue Comments**.

1.  **Open an Issue** (e.g., "Fix login page typo").
2.  **Comment** on the issue:
    ```text
    /agent Please change the login button color to #FF5733 and fix the typo in the header.
    ```
3.  **Wait**:
    -   Go to the **Actions** tab to watch it run.
    -   In about 30-60 seconds, the Agent will:
        -   Analyze your codebase.
        -   Write the code.
        -   **Create a Pull Request** with the fix.

### Best Practices

-   **Be Specific**: "/agent fix it" is bad. "/agent in `index.html`, change the title to 'Welcome'" is good.
-   **Small Tasks**: The agent works best on isolated tasks (one file or small refactors).
-   **Review Code**: Always review the PR generated by the bot before merging.

## ðŸ”§ Troubleshooting

| Problem | Solution |
| :--- | :--- |
| **Action doesn't trigger** | Ensure `.github/workflows/cloud-agent.yml` exists in the **default branch** (main). |
| **"Resource not accessible by integration"** | Check "Workflow permissions" in Settings (Step 3). |
| **Agent fails to write code** | Check the Action logs. It might be an API Key error or the LLM hallucinated a bad file path. |
